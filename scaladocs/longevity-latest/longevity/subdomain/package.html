  1. trait Assoc[P <: Persistent] extends PRef[P]

    a unidirectional association, or "assoc", to a persistent object.

    a unidirectional association, or "assoc", to a persistent object. the left side of the association - that is, the holder of the Assoc instance - is known as the associator. the right side of the association is the associatee.

    there are basically two kinds of associations that you have to understand. the main kind is a "persisted assoc", which means an association to a persistent object that has already been persisted. the associated object can be retrieved using repository method retrieve(Assoc[R]).

    the second kind of association is an "unpersisted assoc", which you can use when creating multiple persistent objects at once with longevity.persistence.RepoPool.createMany. unpersisted assocs are intended to support this single use-case. in general we work with persisted assocs.

  4. trait PRef[P <: Persistent] extends AnyRef

    an indirect reference to a persistent object.

    an indirect reference to a persistent object. they can typically only be dereferenced via the persistence layer.

    there are two kinds of persistent refs: associations and key values. we hope to integrate these two types more in the future. in particular, it should be easier to embed a key value of another aggregate in an entity, in place of embedding an association.
